# Telegram 表情包工作室（项目书）

## 1. 项目概述

`Telegram 表情包工作室` 是一个面向中文用户的 Telegram 机器人系统，支持：
- 一键克隆公开表情包到用户自己的包
- 图片自动制作贴纸（支持连发、自动入包）
- 自定义署名、水印、视觉模式、适配模式
- 可视化个人中心与管理员后台
- 邀请裂变与额度策略（克隆/制作次数）

项目定位是“比常见贴纸机器人更完整、更可运营”的一体化方案：`用户端体验 + 管理端运营 + 稳定运行能力`。

## 2. 目标用户

- 普通用户：想快速克隆/制作贴纸，不想记复杂命令
- 社群运营者：希望通过邀请机制做增长
- 机器人维护者：需要后台查看数据、调额度、群发通知

## 3. 核心能力

### 3.1 用户端
- 粘贴 `t.me/addstickers/...` 自动克隆
- 发送图片自动制作贴纸
- 支持“加入当前包 / 新建包”按钮选择
- 个人中心可视化：额度、最近记录、最近包、默认设置
- 可配置默认模式（视觉、适配、克隆策略、发图去向）

### 3.2 管理端
- 全局统计、运行健康、审计日志
- 用户列表分页、搜索、详情查看
- 按用户调额（加减/直接设置）
- 外链设置（交流群/作者联系方式）
- 群发通知
- 用户 CSV 导出

### 3.3 运行与稳定性
- 轮询模式 / Webhook 模式
- 网络异常重试与错误提示
- 速率限制保护
- 本地 JSON 持久化 + 可选 SQLite

## 4. 技术方案

- 语言：Python 3.10+
- 依赖：`aiohttp`、`Pillow`、`python-dotenv`、`rich`、`qrcode`
- 核心入口：`sticker_studio.py`
- 测试：`test_sticker_studio.py`
- 配置：`.env`

## 5. 目录说明

- `sticker_studio.py`：主程序（CLI + Bot 服务）
- `README.md`：使用文档（中文）
- `PROJECT_BOOK.md`：项目书（本文件）
- `requirements.txt`：依赖清单
- `.env.example`：示例环境变量
- `test_sticker_studio.py`：基础测试

## 6. 部署方式

### 6.1 本地启动
```bash
python -m venv .venv
.venv\\Scripts\\activate
pip install -r requirements.txt
python sticker_studio.py serve
```

### 6.2 环境变量
```env
BOT_TOKEN=123456:your-bot-token
ADMIN_USER_IDS=123456789
BOT_SERVE_MODE=poll
```

## 7. 风险与合规

- 仅可用于有授权素材，禁止侵权传播
- 不可冒充他人官方账号或品牌
- 公开仓库严禁提交真实 Token（`.env` 必须忽略）
